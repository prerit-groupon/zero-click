# Functional Bet 14: Core Service Modernization (Order 2.0)

> **Type:** Functional Bet
> **Org:** Engineering
> **Sponsor:** Ales Drabek
> **DRI:** Abhishek Gupta

---

## Goal

Rebuild Groupon's core order-processing system from a fragile, legacy monolith into a modern,
modular platform that ensures PCI-DSS compliance, reduces operational risk, and unlocks
long-term scalability.

---

## Summary

Using the GROW² framework, Engineering is re-platforming end-of-life components:

| Component | Legacy | Target |
|-----------|--------|--------|
| Ruby | 2.4.6 | Current LTS |
| Rails | 3.2 | Modern Rails |
| OS | Debian 10 | Supported OS |

Key deliverables:
- Modular domain architecture
- Real-time observability
- AI-assisted code development
- Full PCI-DSS compliance

---

## Results

| Metric | Target |
|--------|--------|
| Engineering support time | 50% reduction (200 → 100 hours/year) |
| Customer tickets from order errors | 30% fewer |
